# v2 알려진 한계 (Known Limitations)

이 문서는 변환기의 단순화 지점과, 구조적 제약으로 완전 지원을 미룬 항목을 정리합니다.
손실은 모두 `LossReport`로 기록되므로 다운스트림에서 감지·분기할 수 있습니다.

## 무손실 보강 (텍스트 구제 / 감사 기록)

"조용한 손실"이었으나 무손실 보강 또는 감사 기록 추가로 개선된 항목입니다.

- **하이퍼링크 표시 텍스트 (HWP3)** — `Control::Hyperlink`는 표시 텍스트를 버리지
  않고 평문 블록으로 구제합니다.
- **덧말(Ruby) / 글자겹침(CharOverlap)** — 텍스트/글자를 평문 블록으로 구제합니다.
- **개요 수준 7+ 클램프** — 6으로 평탄화될 때 `LossEntry`를 남깁니다.
- **폰트 손실 보고** — 7개 언어 슬롯(한글/영문/한자/일어/기타/기호/사용자)을 모두
  검사합니다.

## v2에서 구현된 항목

### 인라인 하이퍼링크 (`<href>`)

`Inline::Href { uri, content }`를 추가하고, `Paragraph.field_ranges`로 앵커 텍스트
구간을 식별해 `<href uri="…">…</href>`로 emit합니다. URL은 하이퍼링크 필드
(`FieldType::Hyperlink`) / HWP3 `Control::Hyperlink`의 `command`에서 추출합니다(첫
`;` 세그먼트, `\` 이스케이프 해제). Markdown은 `[anchor](uri)`, payload는 `kind:
"href"` + `uri`로 노출됩니다.

- **남은 한계:** HWP3 경로는 rhwp가 URL을 아직 추출하지 않아(`url = ""`) 표시
  텍스트만 구제됩니다. 내부 책갈피 링크는 HWP가 저장한 원시 형태(`?#<id>` 등)를
  그대로 보존합니다.

### 인텍스트 흐름 객체 위치

일반 문단(개요/리스트 아님)에서 표·그림·수식을 `control_text_positions()`의 문자
위치에 맞춰 본문을 분할하고 정확한 위치에 삽입합니다(`extract_inline_segments`).

- **범위:** 인터리빙 대상은 **흐름 객체(표/그림/수식)** 로 한정합니다. 머리말/꼬리말,
  각주/미주, 글상자 구제 텍스트 등 비흐름 객체는 문단 말미에 배치합니다(페이지 furniture를
  본문 중간으로 끌어오지 않기 위함).
- **제외:** 제목/리스트 문단은 분할이 의미가 없어 "텍스트 먼저, 객체는 뒤" 배치를
  유지합니다.

## v2로 남은 항목

### 미주(Endnote) 별도 표현

- **현재 동작:** DocLang 0.6에 `<endnote>`가 없어 `Block::Footnote`로 강등하고
  `LossEntry`를 남깁니다. 풋노트/미주 번호가 단일 순차 카운터로 병합됩니다.
- **방향:** DocLang이 미주 요소를 도입하거나 preserve 경로에서 구분을 복원할 때
  별도 표현으로 분리합니다.

### 좌표(`<location>`) 미부착 영역

- 글상자 내부 콘텐츠, 각주/미주 본문, 셀 내부 객체에는 좌표를 부착하지 않습니다.
  멀티페이지 블록은 첫 등장 페이지 박스만 기록합니다.
- 레이아웃 실패는 이제 `LossReport`로 보고됩니다(`location-unavailable` /
  `location-page-layout-failed`).
- 인텍스트 분할로 한 문단이 여러 텍스트 블록으로 나뉘면 같은 `(section, para)`
  provenance를 공유하므로, 분할된 텍스트 블록들이 동일한 박스로 매핑됩니다.
- 자세한 정책은 [README의 레이아웃 좌표 절](../README.md#레이아웃-좌표---location)을
  참고하세요.
